# fTree 隐私与数据保护说明

## 1. 适用范围

本文说明 fTree 私人空间版在浏览器、服务端和第三方服务之间如何处理数据，以及用户应承担的保管责任。团队协作版如开放使用，应以其单独发布的隐私说明为准。

## 2. 数据类别

家谱档案可能包含：

- 家谱名称、制作者、版本、开山祖和修改时间。
- 成员标识、姓名、性别、在世状态和备注。
- 父母、配偶、子女及关系顺序。
- GPS、出生、去世、昵称和导入资料中的其他元数据。
- 不同家谱之间的关联关系和展示布局。

其中，在世人员姓名、家庭关系、备注和精确位置应视为敏感个人信息。

## 3. 浏览器本地保存

私人空间版默认把完整工作区打包为加密、压缩档案，保存在当前浏览器的 IndexedDB 中。服务没有注册账户，因此不能把不同浏览器自动识别为同一用户。

以下操作可能导致本地数据丢失：

- 清除站点数据或浏览器历史。
- 使用无痕或访客模式。
- 删除浏览器配置文件。
- 浏览器、磁盘或操作系统故障。
- 设备遗失、损坏或重装。

用户应自行建立可验证的外部备份。

## 4. 服务端处理

正式网站由 Cloudflare Worker 和静态资源服务提供。应用设计中没有用户数据库、KV、对象存储或服务器用户文件目录。

服务端长期配置包括：

- Google、GitHub、GitLab OAuth 应用的 Client ID 和 Client Secret。
- 用于加密 OAuth Cookie 的服务密钥。
- 正式网站地址和静态网页资源。

单次同步请求中，Worker 会临时处理：

- OAuth 授权码、PKCE 校验信息。
- 当前用户的 Access Token、Refresh Token 和有效期。
- 已加密的 `.tree` 正文。
- 第三方文件、Gist 或 Snippet 的标识和更新时间。

应用代码不把这些用户数据写入持久化存储，也不主动记录正文日志。Cloudflare、网络运营商和第三方服务可能依据其自身条款处理请求元数据；应用无法代表这些平台承诺零日志。

## 5. OAuth 凭据

OAuth Client Secret 表示 fTree 应用身份，不用于区分具体用户。用户身份由第三方签发的 OAuth Token 隐式确定。

Token 经服务密钥加密后保存在用户浏览器的 HttpOnly、Secure、SameSite Cookie 中。每个浏览器、每个云服务分别保存凭据。同一浏览器配置文件对同一服务重新授权其他账户时，原凭据可能被覆盖。

退出操作只清除当前浏览器 Cookie：

- 不自动撤销第三方账户授权。
- 不自动删除第三方云端档案。
- 不影响其他设备上的授权 Cookie。

彻底退出时，应在第三方账户安全设置中撤销授权并检查云端副本。

## 6. Google Drive

应用请求 `drive.appdata` 权限，将 `default.tree` 写入用户 Google Drive 的隐藏应用数据目录。

特点：

- 与普通“我的云端硬盘”文件分离。
- 通常不在普通 Drive 页面直接显示。
- 只供获得相应应用授权的请求访问。
- 撤销授权与删除应用数据是两个不同操作。

Google 仍会根据其账户、OAuth 和 Drive 服务规则处理相关数据。

## 7. GitHub

应用请求 `gist` 权限，并建立包含 `default.tree` 的 Secret Gist。

必须理解：

- Secret Gist 不会正常进入公开发现和搜索。
- Secret Gist 并非严格私有；知道 URL 的人可能访问。
- `gist` 权限允许应用代表用户读取和写入 Gist，范围不只是一份备份。
- 应用会临时列出 Gist 元数据以寻找匹配档案，但不在服务端保存列表。

不要公开 Gist URL。涉及在世人员、精确地址或敏感备注的档案，建议选择更严格的存储方式。

## 8. GitLab

应用使用 GitLab OAuth `api` 权限，通过 Snippet API 建立 Private Snippet。

特点：

- 代码把备份设置为 Private。
- 私人 Snippet 由账户持有人控制。
- `api` 是较宽的权限；当前功能只调用 Snippet 相关接口，但 Token 本身权限大于单一文件访问。

用户应启用双重验证，定期审查已授权应用，并确认 Snippet 可见性没有改变。

## 9. 加密边界

`.tree` 使用 gzip 压缩、PBKDF2-SHA-256 派生密钥和 AES-256-GCM 加密。每份档案带有随机盐和随机 IV。

但当前版本没有用户密码输入，使用程序内置的固定口令。这意味着：

- 第三方云服务通常看到的是加密正文，而非直接可读 JSON。
- 掌握程序实现和档案的人仍可能解密。
- 该机制可以防止随手查看，不应当作高强度端到端保密方案。
- 不能依赖此加密安全地公开 Gist URL 或备份下载链接。

## 10. 额外第三方请求

除云同步外，页面功能还可能访问其他第三方服务：

- 成员信息卡片使用头像生成服务，成员姓名可能出现在头像请求 URL 中并发送给该服务。
- 地图使用第三方地图瓦片，第三方可能获得 IP、浏览器信息、地图层级和查看区域。
- 浏览器定位由用户授权；精确 GPS 一旦保存，会进入本地和后续云端备份。

对隐私要求较高时，应避免打开带真实姓名的头像视图、避免保存精确 GPS，并使用网络层拦截策略限制非必要第三方请求。

## 11. 用户责任

用户应当：

- 仅录入实现家谱目的所必需的信息。
- 录入在世人员信息前取得适当同意。
- 不在备注中记录密码、完整证件号、医疗资料或敏感评价。
- 分享图片或档案前核对所有在世人员信息。
- 保护浏览器账户、设备登录、云服务账户和双重验证设备。
- 定期下载并验证备份。
- 删除资料时同步检查浏览器、离线副本和所有第三方云端。

## 12. 删除与保留

由于没有服务端用户数据库，应用无法代替用户统一删除所有副本。删除应分别进行：

1. 在网站内删除不需要的家谱或成员。
2. 清除浏览器站点数据前确认已有备份。
3. 删除 Google Drive 应用数据、GitHub Gist 或 GitLab Snippet。
4. 清理下载目录、移动介质和历史备份。
5. 在第三方账户中撤销 OAuth 授权。

每一步都可能不可逆，操作前应确认保留政策和家庭成员意见。

## 13. 安全事件

发现以下情况时，应立即处理：

- `.tree` 或 Gist URL 被发到公共渠道。
- OAuth Token、Cookie 或 Client Secret 泄露。
- 云端备份被异常修改。
- 浏览器或设备被盗。
- 第三方账户出现未知登录。

处理顺序：断开设备网络、保留必要证据、撤销第三方授权、修改账户密码、启用或重置双重验证、检查并删除暴露副本、从可信备份恢复。报告问题时不得再次上传真实档案。

## 14. 联系与反馈

隐私说明和安全使用建议通过 <https://github.com/mulspace/ftree/wiki> 发布。公开反馈中不得包含真实家谱数据或任何凭据。
