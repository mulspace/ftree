# fTree 完整用户操作指南

本文说明私人空间版的日常操作。网站地址：<https://ftree.dpdns.org>。

## 1. 运行环境

建议使用近期版本的 Chrome、Edge、Firefox 或 Safari，并允许以下能力：

- 站点本地存储，用于保存家谱档案。
- 弹出窗口，用于 Google、GitHub、GitLab OAuth 授权。
- 文件下载和上传，用于 `.tree` 备份。
- 浏览器定位，仅在用户主动使用地图定位时需要。

不建议在无痕窗口、访客模式、自动清理站点数据的浏览器或公共电脑上长期维护家谱。

## 2. 首次进入

首次进入时，系统会在浏览器中建立本地工作区，并加载西汉、东汉和明朝示例。示例用于演示：

- 多家谱管理。
- 成员及亲属关系。
- 不同家谱之间的成员关联。
- 向上溯源和向下继承展示。

示例不是用户的私人资料。可以直接修改，也可以新建家谱后删除示例。

首次进入后建议立即执行一次“打包下载”，确认浏览器允许文件下载，并了解备份保存位置。

## 3. 新建家谱

1. 在左侧“家谱树”区域点击“新建家谱”。
2. 系统建立一个带有“始祖”成员的新家谱。
3. 点击顶部“编辑信息”。
4. 填写：
   - 堂号：用于识别家谱。
   - 开山祖：当前家谱的主节点。
   - 制作者：建议填写维护人或整理小组名称。
   - 版本：建议使用日期或版本号，例如 `2026.07`。
5. 点击“保存信息”。
6. 完成基础资料后立即下载备份。

堂号和制作者应避免包含不必要的身份证号、联系方式或其他敏感信息。

## 4. 编辑成员资料

1. 点击家谱图中的成员节点。
2. 在信息卡片中点击“修改”。
3. 可维护：
   - 姓名。
   - 性别。
   - 在世或已故状态。
   - 备注。
4. 点击“保存成员”。

注意：

- 用于跨家谱关联的成员不能直接改名或删除，应先解除家谱关联。
- 开山祖不能直接删除，应先在“编辑信息”中指定其他开山祖。
- 备注中不要记录账户密码、证件完整号码、医疗隐私或未经同意的敏感评价。

## 5. 添加和维护亲属关系

### 5.1 进入关系编辑

点击成员节点，在信息卡片中选择“添加亲属”或“编辑此人关系”。关系编辑图会显示当前成员、父母、配偶、子女及新增入口。

### 5.2 添加父母

1. 点击“父亲”或“母亲”的新增节点。
2. 同时填写父亲和母亲姓名。
3. 保存后检查配偶关系和子女连线是否正确。

如果只掌握单方资料，应先确认当前版本的录入规则是否允许留空，不要使用猜测姓名代替未知资料。

### 5.3 添加配偶

1. 点击配偶新增节点。
2. 输入姓名并选择性别。
3. 保存后再通过该配偶关系添加共同子女。

### 5.4 添加子女

1. 在对应父母组合下点击“子女”新增节点。
2. 输入姓名和性别。
3. 保存后检查父母组合是否正确。

### 5.5 子女顺序

打开父母关系的子女列表，可将已有成员加入子女列表并调整显示顺序。顺序主要影响家谱布局，不应替代对实际出生顺序的文字说明。

### 5.6 删除成员

删除前系统会提示可能受影响的子女和配偶关系。删除操作可能同时移除直接子女或关系引用，且无法自动撤销。执行前必须先下载备份。

## 6. 家谱查看与导航

- 点击成员节点查看基本资料。
- 选择“导航到”可从该成员重新查看当前家谱分支。
- “向上溯源”控制通过关联家谱向上追踪的最大世代。
- “向下继承”控制通过关联家谱使用目标分支的最大世代。
- “高亮共享”用于识别来自关联家谱的只读成员。

这些展示选项不会改变实际亲属关系，但下载备份前仍应检查当前家谱是否选择正确。

## 7. 关联多个家谱

家谱关联用于连接不同家谱中的同一个历史或家族成员。

1. 打开“家谱关联”页面。
2. 选择两个需要关联的家谱。
3. 选择两边姓名和性别一致的成员。
4. 保存关联。
5. 返回家谱展示，使用溯源或继承深度验证结果。

关联后，相关成员会进入保护状态，避免误改姓名或删除。解除关联前，应确认其他家谱没有依赖该连接。

## 8. GPS 地图

1. 打开“GPS 地图”。
2. 在地图上选择位置，或使用浏览器定位找到附近区域。
3. 选择需要设置位置的成员。
4. 保存位置。
5. 点击地图标记查看成员信息或导航到该成员。

隐私建议：

- 在世人员通常只记录村、镇或城市级别位置，不记录住宅精确坐标。
- 不要把在世人员 GPS 与公开导出的家谱图片同时传播。
- 浏览器定位权限应仅在需要时授权，用完可在浏览器站点设置中关闭。
- 地图瓦片由第三方地图服务提供，对方可能接收到 IP、浏览器信息和地图查看区域。

## 9. 导出图片

1. 进入需要导出的家谱。
2. 调整导航成员、溯源深度、继承深度和高亮状态。
3. 点击“导出图片”。
4. 检查导出结果中的姓名、备注和关联分支。

图片不具备 `.tree` 备份的恢复能力。图片适合查看，`.tree` 才用于恢复编辑数据。

分享图片前，应遮挡或删除在世人员敏感信息，并获得必要授权。

## 10. 下载本地备份

1. 点击“打包下载”。
2. 浏览器生成 `.tree` 文件。
3. 将文件移动到明确的备份目录。
4. 建议复制一份到离线存储。
5. 记录备份日期和维护人。

`.tree` 包含整个工作区，而不是仅当前显示的家谱，包括成员、关系、备注、GPS 和家谱关联。

## 11. 从 `.tree` 恢复

1. 恢复前先下载当前浏览器数据作为回退备份。
2. 点击“上传恢复”。
3. 选择可信来源的 `.tree` 文件。
4. 比较修改时间、家谱数量和成员数量。
5. 确认后执行恢复。
6. 逐个检查新增、覆盖或忽略的家谱。
7. 验证成员、关系、GPS 和家谱关联。
8. 验证完成后再下载一份新备份。

不要导入来源不明的备份，也不要在真实资料环境中测试陌生文件。

## 12. Google Drive 同步

1. 点击“同步全部家谱”。
2. 选择 Google Drive。
3. 选择上传本地当前版本或下载云端当前版本。
4. 在弹出窗口中选择 Google 账户并授权。
5. 比较本地与云端摘要。
6. 确认同步方向。

档案保存到 Google Drive 的隐藏应用数据目录，通常不会显示在普通云盘文件列表中。退出 fTree 授权不会自动删除该文件；可在 Google 账户的第三方应用和应用数据管理中撤销授权或清理数据。

## 13. GitHub 同步

1. 在同步窗口选择 GitHub。
2. 完成 OAuth 授权。
3. 上传后，账户中会建立一个带 `default.tree` 的 Secret Gist。
4. 下载时，系统使用最近更新的匹配 Gist。

GitHub Secret Gist 不是严格私有：它不会正常出现在公开搜索中，但获得链接的人可能访问内容。即使正文是加密档案，也不应把 Gist URL 发到群聊、工单、截图或公开页面。高度敏感的数据不建议使用此选项。

## 14. GitLab 同步

1. 在同步窗口选择 GitLab。
2. 完成 OAuth 授权。
3. 上传后，账户中会建立一个 Private Snippet。
4. 下载时，系统使用最近更新的匹配 Snippet。

应定期检查 Snippet 的可见性仍为 Private，并启用 GitLab 账户双重验证。

## 15. 更换设备

推荐流程：

1. 在旧设备下载最新 `.tree`。
2. 将备份通过安全介质传到新设备。
3. 在新设备打开网站并先下载新设备的初始数据。
4. 使用“上传恢复”导入 `.tree`，或授权同一云账户并下载云端版本。
5. 核对家谱数、成员数和最近修改时间。
6. 确认无误后再清理旧设备。

不要仅凭“云端同步成功”提示就立即删除旧设备数据。

## 16. 退出和撤销授权

- 退出云服务会清除当前浏览器的授权 Cookie。
- 清除 Cookie 不会删除第三方云端档案。
- 如需彻底终止访问，应同时在 Google、GitHub 或 GitLab 账户中撤销应用授权。
- 如需删除云端备份，应在相应云服务中删除应用数据、Gist 或 Snippet。

## 17. 常见问题

### 页面重新打开后资料消失

检查是否使用了无痕模式、是否切换浏览器配置文件、是否清理过站点数据。若有 `.tree` 或云端副本，按恢复流程操作。

### OAuth 窗口打不开

允许本站弹出窗口，关闭过度严格的脚本拦截扩展后重试。确认访问的是正式 HTTPS 网站。

### 同步方向不确定

先取消同步，分别记录本地和云端的修改时间、家谱数、成员数，再下载当前本地备份。不要依靠文件名判断新旧。

### 出现多个同名 Gist、Snippet 或文件

系统通常使用最近更新的一份。确认内容后在第三方账户中手动清理旧的重复项，清理前先下载备份。

### 备份无法恢复

确认文件没有被聊天软件、编辑器或网盘预览功能修改；重新从原位置下载。如果所有副本均损坏，服务端没有用户数据库可用于恢复。

## 18. 报告问题

请按事项类型选择入口：

- 新功能和改进建议：提交[需求 Issue](https://github.com/mulspace/ftree/issues/new?template=feature_request.yml)。
- 页面错误、数据异常、同步或恢复故障：提交[问题 Issue](https://github.com/mulspace/ftree/issues/new?template=bug_report.yml)。
- 使用咨询、资料整理方法和其他非故障事项：在 Wiki [其他列表](https://github.com/mulspace/ftree/wiki/其他列表)登记。

问题报告必须写清环境、前置条件、完整复现步骤、实际结果、期望结果、出现频率、影响程度和已经尝试的方法。可以包含：

- 浏览器名称和版本。
- 操作系统。
- 复现步骤。
- 不含私人资料的截图。
- 错误提示文字。

不得提交真实 `.tree`、Cookie、OAuth Token、Client Secret、完整成员名单、私人备注或精确 GPS。完整规范见[支持与反馈](../SUPPORT.md)。
